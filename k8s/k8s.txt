==========

Control Plane COmponents : 

1. API Server (kube-apiserver)
--> Front door of K8s - All communication goes through it.
--> We can use "kubectl" to talk to the API server.
--> Validates and processes all requests.
--> Only component that can talks to etcd directly.

2. etcd
--> Database of K8s - THis contains all cluster data.
--> key-value (tags name=webserver)
--> Stores "desired state" and "current state" of our workload
--> if etcd is lost, the cluster state is lost.

3. Controller Manager (kube-controller-manager)
Act as a Supervisor - Monitors etcd and ensures desired state matches current state.
--> If we ask for 3 replicas and only 2 are running.. CM creates the 3rd replica.
--> Manages multiple controllers : replicaset, Deployment, Node, Job controllers

4. Scheduler (kube-scheduler)
--> Decision Maker. THis decides which worker node should run new pod.
--> Its checks the resource availability, constraints and tains
--> This DOES NOT run the pod, It just picks the best node to run the pod.

5. CCM - Cloud Controller Manager
--> Connects K8s to cloud provider services
--> Without CCM, K8s cant use ALB / EBS Volumes / Cloudwatch.


WORKER NODE / DATA PLANE

1. kubelet
--> This is an agent runs on every worker node.
--> Talks to API server to get instructions
--> Ensures pods are running and healthy
--> If a pod crashes, kubelet restarts it automatically.

2. Container Runtime
--> Responsible for pulling images and running containers.
--> K8s supports: containerd, CRI-O
--> Earlier supported docker. Containerd actually uses docker's underlying runtime.
--> Kubelet asks the runtime to pull the image and start the container inside the pod.

3. kube-proxy
--> Networking component on every node.
--> Manages networking rules and pod-to-pod communication
--> Assigns a unique IP address to every pod.
--> Routes traffic to the correct pod behind a service.


===========

EKS : Elastic Kubernetes Service : THis is a fully manged K8s service provided by AWS.
AWS Manages control plane components. AWS Maintains 3 copies of control plane components across 3 AZs.
--> We dont get acess to control plane nodes.
--> We can integrate with AWS services i.e; IAM (IRSA), VPC, ALB, Route53, EBS, Cloudwatch

To run our workload, we have multiple options, listed below:

EC2 Self Managed : WE provision ec2 instances, WE manage updates, patches, WE will have full control

EC2 Managed Node Group : AWS provision ec2 instances, AWS manage updates, patches, WE will have little limited control... We can choose required configuration.. MOST COMMON CHOICE

EKS Auto Mode : AWS Manages Compute, Network, Storage.. 

AWS Fargate : Serverless, No nodes at all.. Directly deploy pods.. Pay-per-pod..

Prerequisites:

1. AWS CLI install and configure (https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)
2. Install kubectl in your laptop (https://kubernetes.io/docs/tasks/tools/)
3. Install eksctl (https://docs.aws.amazon.com/eks/latest/eksctl/installation.html)


eksctl create cluster --name=ekswithavinash --version 1.34 --region ap-south-1 --zones=ap-south-1a,ap-south-1b --nodegroup-name mynodegroup --node-type c7i-flex.large --nodes 2 --managed

eksctl create cluster --name=ekswithavinash --version 1.34 --region ap-south-1 --zones=ap-south-1a,ap-south-1b
 --nodegroup-name mynodegroup --node-type c7i-flex.large --nodes 2 --node-ami-family=AmazonLinux2023 --managed


OIDC Provider: OpenID Connect allows AWS IAM to authenticate with K8s service accounts. 

eksctl utils associate-iam-oidc-provider --region ap-south-1 --cluster ekswithavinash --approve
